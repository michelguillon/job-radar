## What to build

### 1. `cli/digest.py`

The morning review tool. Reads the latest scored records, joins with
workflow state from `corpus/activity_log.jsonl`, and surfaces what's new.

**Behaviour:**
- Default: show all records scored since the last digest run
- Cursor: persist last-run timestamp to `corpus/.digest_last_run`
  (gitignored). Default `--since` = timestamp of last successful digest
  run, falling back to 24 hours if no previous run exists.
  Use **start** timestamp (not finish) — same reasoning as collect cursor.
- Filter by `--min-fit` (default 6) — only show fit_score ≥ N
- Filter by `--since` override — explicit ISO datetime or "yesterday"
- Output: a readable table sorted by priority_score desc:
  company | role title | fit_label | fit_score | location | source_url
- Optional `--export` flag: write the same table to
  `corpus/digest_{YYYYMMDD}.md`
- Show count summary at top: "N new roles since {since}"
- Exclude roles already tracked with status ≠ "new" (already in pipeline)
  unless `--all` flag is passed

**CLI:**
```bash
python -m cli.digest                          # since last run, min-fit 6
python -m cli.digest --min-fit 7              # higher bar
python -m cli.digest --since 2026-06-09       # explicit date
python -m cli.digest --since yesterday        # convenience
python -m cli.digest --export                 # write to corpus/digest_{date}.md
python -m cli.digest --all                    # include already-tracked roles
```

**Tests:** `tests/test_digest.py`
- since-cursor logic (last run timestamp → correct window)
- min-fit filter
- excludes already-tracked roles by default
- --all shows everything
- export writes correct file
- injectable clock + IO (same pattern as track.py)

### 2. `cron/collect_weekly.sh`

Shell wrapper for the weekly collection run.

```bash
#!/usr/bin/env bash
# Weekly collection — runs every Sunday at 08:00
# Cron: 0 8 * * 0 /home/michel/dev/job-radar/cron/collect_weekly.sh
set -euo pipefail
cd /home/michel/dev/job-radar
docker compose run --rm job-radar python -m cli.collect --source all
docker compose run --rm job-radar python -m cli.dedupe
docker compose run --rm job-radar python -m cli.prefilter
docker compose run --rm job-radar python -m cli.label
docker compose run --rm job-radar python -m cli.validate
docker compose run --rm job-radar python -m cli.score
docker compose run --rm job-radar python -m cli.stats --export-index
```

Log output to `/var/log/job-radar/collect_weekly.log` with timestamps.
Create the log directory if it doesn't exist.

### 3. `cron/digest_daily.sh`

Shell wrapper for the daily morning digest.

```bash
#!/usr/bin/env bash
# Daily digest — runs weekdays at 07:30
# Cron: 30 7 * * 1-5 /home/michel/dev/job-radar/cron/digest_daily.sh
set -euo pipefail
cd /home/michel/dev/job-radar
docker compose run --rm job-radar python -m cli.digest --min-fit 6 --export
```

Log output to `/var/log/job-radar/digest_daily.log`.
